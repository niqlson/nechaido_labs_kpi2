# Online Shop - Use Case Documentation

## Опис проекту
Ця лабораторна робота представляє модель варіантів використання (Use Case Model) для системи електронної комерції. Модель включає діаграму варіантів використання, список вимог та матрицю трасування.

## Компоненти системи

### Актори
- **User** (базовий актор)
- **Guest** (гість) - успадковує від User
- **Customer** (користувач) - успадковує від User  
- **Admin** (адміністратор) - успадковує від User

### Основні функціональні групи
1. **Управління користувачами** - реєстрація, авторизація
2. **Каталог товарів** - перегляд, пошук, деталі товарів
3. **Кошик** - додавання, редагування, видалення товарів
4. **Замовлення** - оформлення, оплата, відстеження
5. **Відгуки** - написання, редагування, видалення
6. **Бронювання** - створення, скасування, модерація
7. **Адміністрування** - управління товарами, знижками, замовленнями

## Нефункціональні аспекти
Проект включає критичні нефункціональні вимоги, що безпосередньо випливають з Use Case діаграми:
- **Безпека** автентифікації та платіжних операцій
- **Надійність** транзакцій замовлень та кошика
- **Зручність** процесу оформлення та управління контентом

## Вміст лабораторної роботи
- `usecase.png` - графічне представлення діаграми
- `usecase.txt` - код діаграми на PlantUML
- `requirements.md` - список функціональних та нефункціональних вимог
- `traceability-matrix.md` - матриця трасування вимог до Use Cases

## Вимоги до перегляду
- Для перегляду `.txt` файлів: PlantUML або онлайн-редактор
- Для перегляду графічної діаграми: будь-який переглядач зображень
- Для перегляду документації: текстовий редактор з підтримкою Markdown

## Валідація моделі
- ✅ Використано 2+ типи акторів (людина + зовнішня система)
- ✅ Застосовано асоціації, include/extend, узагальнення
- ✅ Всі функціональні вимоги покрито Use Cases
- ✅ Критичні нефункціональні вимоги пов'язані з Use Cases
- ✅ Матриця трасування містить зв'язки "багато-до-багатьох"
